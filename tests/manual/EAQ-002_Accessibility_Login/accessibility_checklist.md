# Чек-лист тестирования доступности для страницы регистрации

## Клавиатурная навигация
- [ ] Ссылка «Перейти к содержимому» (skip to content) доступна первой
- [ ] Все поля ввода (`<input>`) доступны через `Tab`
- [ ] Все ссылки (`<a href>`) доступны через `Tab`
- [ ] Все кнопки (`<button>`) доступны через `Tab`
- [ ] Фокус виден на всех интерактивных элементах
- [ ] Клавиша `Enter` активирует кнопки и ссылки
- [ ] Клавиша `Space` активирует чекбоксы и радиокнопки
- [ ] Выпадающие списки управляются стрелками
- [ ] Клавиша `Esc` закрывает модальные окна и выпадающие списки

## Скринридер (NVDA / JAWS)
- [ ] Заголовок страницы (`<title>`) объявлен первым
- [ ] Структура заголовков (`h1` → `h2` → `h3`) логична
- [ ] Все поля формы имеют связанные `<label>` (название поля озвучивается)
- [ ] Сообщения об ошибках объявляются автоматически (ARIA live region)
- [ ] Изображения имеют осмысленный `alt`-текст
- [ ] ARIA-атрибуты (`role`, `aria-label`) используются корректно

## Контрастность
- [ ] Обычный текст (основной) – контрастность ≥ 4.5:1
- [ ] Крупный текст (заголовки) – контрастность ≥ 3:1
- [ ] Кнопки, границы полей ввода – контрастность ≥ 3:1
- [ ] Состояния `:hover`, `:focus`, `:active` сохраняют контрастность

## Инструменты автоматической проверки
- [ ] **WAVE:** нет критических ошибок (красные иконки)
- [ ] **Lighthouse:** Accessibility Score ≥ 90
- [ ] **axe DevTools:** нет нарушений уровня A / AA

## Дополнительные проверки
- [ ] Масштабирование до 200%: контент не обрезается, нет горизонтальной прокрутки
- [ ] Адаптивная вёрстка (320px, 768px, 1280px): элементы не накладываются
- [ ] Сообщения об ошибках валидации понятны и подсвечивают проблемное поле
